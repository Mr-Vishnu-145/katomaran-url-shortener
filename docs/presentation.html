🎯 The LinkSphere Solution

LinkSphere is a full-featured, enterprise-grade URL shortening and management application. It bridges the gap between simple link shortening and comprehensive operational business intelligence.

It enables users to generate short codes, define expiry conditions, assign custom aliases, update destinations post-creation, and review deep analytics.

For administrators, it features global dashboard reporting, user control (suspension/soft-deletion), real-time traffic monitoring, and automated AI traffic insights.

🚀 Key Engineering Priorities

  • High-Performance Redirects: Minimizing redirect latencies via server-side memory caching (node-cache).
  • Rich Visit Analytics: Granular logging of device type, browser engine, operating system, geographical country (IP lookup), and referrers.
  • Stateless Security: Dual-token JWT architecture (short-lived access tokens, secure HTTP-only refresh tokens) + rate limiting.
  • Clean Separation of Concerns: A decoupled REST architecture backended by Node/Express and frontended by React/Vite.
Architectural Choice: Decoupled Client-Server architecture allows frontend and backend to scale independently. Node.js's event loop naturally supports the highly concurrent request profile generated by redirection traffic. PostgreSQL provides acid compliance for links and audit trails while in-memory cache saves database reads.

Frontend Client

React (Vite-powered): Ultra-fast compilation and HMR.

Zustand: Minimalist state management for auth & theme persistence.

TanStack Query: Handles client cache, network fetching, and auto-refetch validation.

Recharts & Framer Motion: Premium dashboard visualizations and micro-animations.

API Server

Express (Node.js): Lightweight, middleware-centric HTTP handler.

Socket.IO: Bi-directional persistent WebSockets for live analytics streaming to admin dashboards.

Node Cache: Internal in-memory engine storing redirects, avoiding heavy SQL joins on redirect hits.

UA-Parser-JS: Translates HTTP headers into granular browser, OS, and device metrics.

Database & AI

PostgreSQL: Relational persistence engine with custom indexes for fast lookup.

Pool Client (pg): Reuses TCP sockets, avoiding connection handshakes.

Gemini Pro API: Reads anonymized analytics aggregates and produces human-digestible charts summaries and click recommendations.

Why not an ORM? The backend uses raw SQL queries via PostgreSQL pg.Pool. This eliminates the query abstraction overhead of ORMs like Sequelize or Prisma, allowing precision index tuning (e.g., partial index on active URLs) and faster JSON generation directly in SQL.

PostgreSQL Table Schemas

users urls url_visits refresh_tokens password_reset_tokens deleted_users

Database Tuning (Indexes)

To guarantee performance under load, critical lookup columns use explicit B-Tree indexing:

  1. idx_users_email: Accelerates registration and login verification.
  2. idx_urls_short_code: Supports rapid query by redirect route.
  3. idx_url_visits_url_id: Enhances charts aggregation performance.
  4. idx_urls_active: A partial filter index (WHERE is_active = true) to minimize search trees for valid redirect links.
Referential Integrity: ON DELETE CASCADE is assigned to user references, ensuring that when an account is deleted (or completely expunged), associated URLs, visits, and tokens are cleaned up instantly by the RDBMS engine.

⚙️ Modular MVC Pattern

The backend implements a clean Model-View-Controller (MVC) architecture, tailored for APIs:

  • Routes Layer: Express Routers receive the HTTP endpoints and delegate calls to the correct controller actions.
  • Middleware Layer: Handles authentication, verification, rate limiting, and global error parsing before requests hit the business logic.
  • Controllers Layer: Core business logic. Communicates directly with PostgreSQL via parameters queries or utilizes Node Cache.
  • Config Layer: Database pooling, migration triggers, and schema definition.

🔑 Auth & Admin Controllers

auth.js: Handles user operations.

  • Password hashing using bcrypt with 12 salt rounds.
  • Google OAuth validation and user sync.
  • Dual-JWT generation (Access token, httpOnly refresh token).
  • OTP generation and expiration for account updates.

admin.js: Administrative panel endpoints.

  • User listing, suspension toggle, and account purging.
  • Global link management and deletion.
  • Platform-wide summary metrics (total links, visits, growth rate).
  • Access auditing via the deleted_users history log.

🔗 URL & Analytics Controllers

url.js: Link management operations.

  • Generates randomized 6-character alphanumeric shortcodes.
  • Validates custom aliases against database collisions.
  • Applies expiration timestamps.
  • Saves active redirects inside local Node Cache.

analytics.js: Processes telemetry data.

  • Performs SQL aggregations (GROUP BY browser, device, country).
  • Builds time-series data for traffic charts.
  • Coordinates with Google Gemini API to compile textual analysis summaries.

🛡️ Access Control & JWT Validation

Security is structured using layered Express middlewares:

  1. requireAuth: Extracts the JSON Web Token from the HTTP Authorization header (Bearer <token>). If valid, it decodes the payload, checks for account suspension or deletion, and attaches the user data to the request object.
  2. requireAdmin: Runs after authentication validation. Confirms that the decoded user payload has role === 'admin'. If not, it blocks request execution with a 403 Forbidden payload.

🧱 Rate Limiter & Global Error Handler

express-rate-limit: Configured on public-facing endpoints (e.g. login and registration) to prevent credential stuffing. Limit: 5 requests per 15 minutes.

Centralized Error Middleware: Catches unhandled promise rejections. Instead of returning raw Node stack traces which reveal server paths, it converts errors into clean JSON formats:

{
  "status": "error",
  "code": "INTERNAL_SERVER_ERROR",
  "message": "An unexpected error occurred"
}

⚡ Client-to-Server API Calls

Axios client configuration: Serves as the centralized communication bridge between the frontend React shell and backend.

  • Request Interceptor: Monitors Zustand state and injects JWT authorization headers (Bearer token) on every outgoing call transparently.
  • Response Interceptor (JWT Rotation): If an API call fails with a 401 Unauthorized status, it intercepts the stream, sends a request to the backend /refresh route, updates the client tokens, and retries the original request seamlessly.

🌐 Backend Routing & Gateway

Express Server Gateway: Routes client requests and matches prefixes (/api/auth, /api/urls) to route files and controllers.

  • Redirection Gateway: Matches shortcodes via GET /:shortCode, checks local memory cache (NodeCache) to bypass DB delay, returns a 302 Found redirect immediately, and logs visitor analytics asynchronously.
Design Motivation: Interceptors keep security concerns out of individual page components. Redirect handling is prioritized for speed—issuing a 302 header before database logging finishes to reduce latency.

The React application is optimized for speed, code-splitting, and asset delivery via Vite's bundling engine:

🐻 Local State Management (Zustand)

Zustand is used for quick, lightweight global variables instead of heavy React Context wrappers:

  • Session Management: Stores user details, login state, and JWT tokens. Accessible globally in two lines of code without triggering full app re-renders.
  • App Initialization: Controls splash screen rendering during initial auth status checks.
  • Theme Preference: Persists user preference (dark/light) inside local storage and modifies body class lists immediately.

🔄 Server State & Cache Invalidation

TanStack React Query manages all remote HTTP data exchanges, loading indicators, and caching policies:

  • Query Keys: Keeps requests organized. (['urls'], ['analytics', id]).
  • Auto-Invalidation: When a user creates or deletes a URL, query cache keys are invalidated, prompting background updates without reloading.
  • Axios Interceptors: If an API call fails with a 401 Unauthorized status, the interceptor automatically attempts to rotate the access token using the HTTP-Only cookie, repeating the original call transparently.

📡 Real-time Updates (Socket.IO)

LinkSphere keeps administrators informed without manual refresh polling:

  • WebSocket Connection: Built on Socket.IO, the frontend admin console opens a persistent full-duplex TCP stream to the server.
  • Event Emitter: Every time a visitor accesses a shortened code, the redirect router captures analytics, saves to PostgreSQL, and emits a new_visit payload.
  • Reactive UI: The React app catches the event and inserts the record into the active user feed with CSS animations.

🔐 Complete Authentication Flow

1. Client Request

User submits credentials via Login/Register page.

➡️
2. Server Validation

Bcrypt compares hashed credentials or verifies Google OAuth.

➡️
3. Tokens Issuance

Access token is issued; Refresh token is stored in HTTP-only Cookie.

➡️
4. Navigation Guard

Zustand is updated; Route guards open the application Dashboard.

Token Rotation Pattern: Setting access token lifetime to 15 minutes restricts damage from token leakage. If the token expires, the client calls the /refresh endpoint automatically in the background using the secure, httpOnly, sameSite=strict refresh cookie.

🔗 URL Shortener Generation Lifecycle

1. User Submission

User provides original URL, custom alias (optional), and expiration date.

➡️
2. Validation Check

Server checks URL syntax, alias duplication, and expiration limits.

➡️
3. PostgreSQL Store

Stores URL entry with a unique shortcode mapped to the user ID.

➡️
4. Cache Warming

Sets shortcode-to-original mapping inside NodeCache memory.

Validation Rules: The backend verifies that the original URL is validly formatted via URL API validation. If a custom alias is provided, it is alphanumeric parsed to prevent SQL injections and URL formatting issues.

⚡ Redirection Traffic pipeline

1. Visitor Hit

Visitor requests shortened address: /lnkS.

➡️
2. Caching Verification

Checks Node Cache. If cache miss, queries DB and updates cache.

➡️
3. Telemetry Gathering

Asynchronously logs country, agent, browser, OS, and client IP.

➡️
4. Redirect Output

Responds to visitor with an immediate HTTP 302 redirect header.

Performance Design: The redirect router issues the HTTP response header (302 Found) *before* finishing SQL database logs insertion for telemetry. This guarantees that visitors are redirected instantly, and database write operations do not block the user experience.

🛑 Administrative Moderation Actions

LinkSphere provides robust operational administrative controls to manage security threats:

  • User Suspension: Flipping is_suspended = true locks the target user out of their session. The authentication middleware rejects active tokens immediately, blocking unauthorized operations.
  • Link Deactivation: Admins can force deactivate any link. Deactivation instantly wipes the item from the server-side redirection cache.

🗑️ Secure Soft-Deletion Audit Logs

To maintain historical context and support GDPR/compliance policies, account deletion implements a soft-delete mechanism:

  • Deleted User Register: Deletion updates status to deleted. It copies account metadata (excluding hashed passwords) into the deleted_users table.
  • Cascade Purging: This trigger safely disables their active short links and invalidates related active cookies and sessions.
  • Audit logs: Enables compliance monitoring for systems auditing and administrative tracking.

⚙️ Defense-in-Depth Mechanisms

  • Parameterized Database Queries: Avoids inline SQL string concats. Protects database structures from SQL injection attacks.
  • Bcrypt Hashing: Safe, computational-heavy encryption rounds make brute-force attacks infeasible.
  • Security Headers (Helmet): Restricts MIME types, implements XSS protection, and configures Strict-Transport-Security.
  • CORS Whitelisting: Server rejects API access from unauthorized domains.

🍪 Secure JWT Session Architecture

Two-Token authentication pattern:

  1. Access Token: Short lifetime (15 mins), stored in client memory (volatile). Sent in Authorization header. Minimal risk if leaked.
  2. Refresh Token: Long lifetime (7 days), stored in database. Sent only to client via secure, httpOnly, sameSite=strict cookie. Restricted from JavaScript access (preventing XSS access).
  3. Suspension Sync: Both tokens are validated against database suspension records upon each call.

🏁 Project Takeaways

LinkSphere represents a fast, secure, modern platform for handling URL shortener workloads. By separating the client app from the REST API, using cache layers, and supporting real-time event updates, it provides an outstanding developer layout and user experience.

The code is fully structured, index-tuned, and audited for credential vulnerabilities.

📈 Future Scalability Path

  • Distributed Caching (Redis): Transition local Node Cache to a dedicated Redis cluster, allowing scaling across multiple backend instances.
  • Database Read Replicas: Route heavy analytical read queries (Gemini analytics) to PostgreSQL read-replicas while reserving primary database node for writes.
  • Docker Containerization: Containerize frontend and backend to support auto-scaling orchestrations via Kubernetes.
